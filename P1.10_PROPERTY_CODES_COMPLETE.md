## How It Works

### Auto-Generation Logic
When a property is created, a trigger automatically generates a unique code:

**Example Property Names → Codes**:
- "Sunset Apartments" → "SUN-1234"
- "Oak Ridge Towers" → "OAK-9876"
- "123 Main Street" → "123-4567"
- "The Grove" → "THE-0001"

**Generation Algorithm**:
1. Extract first 3 characters from property name (alphanumeric only)
2. Convert to uppercase
3. Pad with 'X' if less than 3 characters
4. Append 4 random digits
5. Check uniqueness, retry up to 10 times
6. Fallback to timestamp if collisions persist

### User Flow

**Landlord Side**:
1. Create property → Code auto-generated (e.g., "SUN-1234")
2. View PropertyDetailScreen → See property code prominently displayed
3. Share code with tenant via text, email, or in person

**Tenant Side**:
1. Open JoinPropertyScreen
2. Enter property code: "SUN-1234"
3. Code converted to uppercase automatically
4. System looks up property by property_code field
5. If not found, falls back to UUID lookup (backward compatibility)
6. Tenant joins property successfully

## Technical Implementation

### Database Migration

**File**: `supabase/migrations/20260213141740_property_codes.sql` (68 lines)

#### 1. Add property_code Column
```sql
alter table public.properties
add column if not exists property_code text unique;
```

#### 2. Code Generation Function
```sql
create or replace function generate_property_code(property_name text)
returns text
language plpgsql
as $$
declare
  prefix text;
  code text;
  code_exists boolean;
begin
  -- Extract first 3 letters (uppercase, alphanumeric only)
  prefix := upper(regexp_replace(
    substring(property_name from 1 for 3),
    '[^A-Za-z0-9]',
    '',
    'g'
  ));

  -- Pad with 'X' if too short
  while length(prefix) < 3 loop
    prefix := prefix || 'X';
  end loop;

  -- Try to generate unique code (max 10 attempts)
  for i in 1..10 loop
    -- Generate 4 random digits
    code := prefix || '-' || lpad(floor(random() * 10000)::text, 4, '0');

    -- Check uniqueness
    select exists(
      select 1 from public.properties
      where property_code = code
    ) into code_exists;

    if not code_exists then
      return code;
    end if;
  end loop;

  -- Fallback: append timestamp
  return prefix || '-' || lpad(extract(epoch from now())::bigint::text, 4, '0');
end;
$$;
```

#### 3. Trigger for Auto-Generation
```sql
create or replace function set_property_code()
returns trigger
language plpgsql
as $$
begin
  -- Only generate if property_code is null
  if new.property_code is null then
    new.property_code := generate_property_code(new.name);
  end if;
  return new;
end;
$$;

create trigger trigger_set_property_code
  before insert on public.properties
  for each row
  execute function set_property_code();
```

#### 4. Backfill Existing Properties
```sql
update public.properties
set property_code = generate_property_code(name)
where property_code is null;
```

#### 5. Index for Fast Lookups
```sql
create index if not exists properties_property_code_idx
on public.properties(property_code);
```

// Look up property by property_code (friendly code like "SUN-1234") or UUID
const code = propertyCode.trim().toUpperCase();

// Try property_code first (friendly code)
let { data: property, error: propError } = await supabase
  .from('properties')
  .select('id, name, address, city, property_code')
  .eq('property_code', code)
  .single();

// If not found by property_code, try by UUID (backward compatibility)
if (propError || !property) {
  const result = await supabase
    .from('properties')
    .select('id, name, address, city, property_code')
    .eq('id', propertyCode.trim())
    .single();

  property = result.data;
  propError = result.error;
}

if (propError || !property) {
  Alert.alert('Error', 'Property not found. Please check the code and try again.');
  return;
}
