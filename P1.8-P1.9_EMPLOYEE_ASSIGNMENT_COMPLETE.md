## What Was Delivered

### Employee Assignment for Maintenance Requests
Landlords and employees can now assign maintenance requests to employees directly from the MaintenanceRequestDetailScreen.

**Features**:
- "Assign Employee" button in MaintenanceRequestDetailScreen
- Employee picker with full name display
- "Unassign" option to remove assignment
- Shows assigned employee name (not just ID)
- Real-time assignment updates with confirmation alerts
- useEmployees hook to fetch all employees from profiles
- Role-based permissions (landlords and employees only)

---

## How It Works

### User Flow
1. **View Maintenance Request**
   - Navigate to MaintenanceRequestDetailScreen from any dashboard
   - See "Assigned To" section showing current assignment status

2. **Assign Employee**
   - Tap "Assign Employee" button (or "Reassign Employee" if already assigned)
   - Alert menu shows list of all employees (sorted by name)
   - Select employee from list
   - Confirmation alert: "Request assigned to [Employee Name]"
   - Screen auto-navigates back to dashboard

3. **View Assignment**
   - "Assigned To" section shows employee's full name
   - If no assignment: "Not assigned yet" (italicized)
   - Employee sees their assigned requests in their dashboard

4. **Unassign Employee**
   - Tap "Reassign Employee" button
   - Select "Unassign" from alert menu
   - Confirmation: "Employee assignment removed"
   - Request returns to unassigned state

### Data Flow
```
Landlord/Employee views request
  ↓
Taps "Assign Employee" button
  ↓
useEmployees hook fetches all employees from profiles table
  ↓
Alert menu displays employee names (full_name or email)
  ↓
User selects employee
  ↓
useUpdateMaintenanceRequest mutation updates assigned_to field
  ↓
Query invalidation refreshes dashboard
  ↓
Assigned employee sees request in their view
```

## Technical Implementation

### New Hook: useEmployees

**Location**: `src/hooks/useData.ts` (lines 389-405)

```typescript
export const useEmployees = () => {
  return useQuery({
    queryKey: ['employees'],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('profiles')
        .select('id, full_name, email, role')
        .eq('role', 'employee')
        .order('full_name', { ascending: true });

      if (error) throw error;
      return data || [];
    },
    staleTime: 5 * 60 * 1000, // Cache for 5 minutes
  });
};
```

**Why profiles table?**
- User roles are stored in profiles table (role field)
- Includes full_name and email for display
- Filters by role = 'employee' to get only employees
- Sorted alphabetically by full_name for easy selection

#### 3. Assignment handler function (lines 82-112)
```typescript
const handleAssignEmployee = async (employeeId: string | null) => {
  if (activeRole !== 'landlord' && activeRole !== 'employee') {
    Alert.alert('Permission Denied', 'Only landlords and employees can assign requests');
    return;
  }

  try {
    setIsUpdating(true);
    await updateRequest.mutateAsync({
      id: request.id,
      assigned_to: employeeId,
    });

    const employee = employees.find((e: any) => e.id === employeeId);
    const message = employeeId
      ? `Request assigned to ${employee?.full_name || 'employee'}`
      : 'Employee assignment removed';

    Alert.alert('Success', message, [
      {
        text: 'OK',
        onPress: () => navigation.goBack(),
      },
    ]);
  } catch (error: any) {
    console.error('Error assigning employee:', error);
    Alert.alert('Error', error.message || 'Failed to assign employee');
  } finally {
    setIsUpdating(false);
  }
};
```

#### 4. Employee picker function (lines 114-142)
```typescript
const promptEmployeeAssignment = () => {
  if (employeesLoading) {
    Alert.alert('Loading', 'Loading employees...');
    return;
  }

  if (employees.length === 0) {
    Alert.alert('No Employees', 'No employees available to assign. Add employees from user management.');
    return;
  }

  const options = [
    ...employees.map((employee: any) => ({
      text: employee.full_name || employee.email,
      onPress: () => handleAssignEmployee(employee.id),
    })),
    {
      text: 'Unassign',
      style: 'destructive' as const,
      onPress: () => handleAssignEmployee(null),
    },
    {
      text: 'Cancel',
      style: 'cancel' as const,
    },
  ];

  Alert.alert('Assign Employee', 'Select an employee to assign this request:', options);
};
```

### Interaction Design
- **Alert Menu**: Native iOS/Android pattern for selection

### Text Display Logic
